MySQL坑:
	尽量避免使用MySQL关键字作为字段名, 比如describe, 这就是数据库系统的字段名, 会有冲突的
	如果实在是冲突了, 把你的字段名加上 `` , 就可以了, MySQL解析器会自动解析成为字段名

CONCAT: 合并的意思, 就是在原来的数据结尾加上字符串
	update goods set `describe` =  CONCAT(`describe`,'qqqWW')  where  goods_id = '9996';

mysql 坑:
	有一天, 我的mysql数据库中有一个表中的字段是 PRIMARY KEY的( 至少desc 表名; 的时候是显示这个字段是PRI的, 那就是PRIMARY KEY喽? )
	但是查了好久才知道 , 其实并不是的...	
	所以一开始就一个劲的用这个语法: 
		alter table `goods` drop PRIMARY KEY; 		-> 意思是取消主键
	但是死活执行不成功, 始终报错~~~
	
	后来网上查了很久, 用 
		show create table 表名;    -> 查出这个字段并不是 PRIMARY KEY 类型的
	而是  UNIQUE KEY 类型的...
	我了个去!!!!!!!!
	坑爹啊!!!!!
	
	所以直接用 
		ALTER TABLE 表名 DROP INDEX 字段名;
	取消了所谓的`主键`

MVC框架中 (SSM框架):
***Mapper.xml文件 不要放在 cn.xxxxx.mapper 目录下面, 因为  idea默认是不编译 src\main\java下的xml文件
xml文件一律放在 resources 目录下面
注意: 在idea编辑器中, 记得把 resource 设置成为Rources Root 文件夹
同时 :
        <!-- 扫描Mapper层的配置文件 -->
        <property name="mapperLocations" value="classpath:/*.xml"/>
